2023-12-25-Mon 访问加速 Web

主要针对由github pages生成的静态博客。

优化国内用户访问速度（当然翻墙就没有这些问题，但还是省点流量好）。

环境

  • github pages
  • Gridea 静态博客框架
  • valine 评论系统

搭建过程

获取APP ID 和 APP Key

请先登录或注册 LeanCloud, 进入控制台后点击左下角创建应用（开发版即可）。

应用创建好以后，进入刚刚创建的应用，选择左下角的设置>应用Key，然后就能看到你的APP ID和APP Key了。

HTML 片段

修改初始化对象中的appId和appKey的值为上面刚刚获取到的值即可(其他可以默认)。

<head>
    ..
    <script src='//unpkg.com/valine/dist/Valine.min.js'></script>
    ...
</head>
<body>
    ...
    <div id="vcomments"></div>
    <script>
        new Valine({
            el: '#vcomments',
            appId: 'Your appId',
            appKey: 'Your appKey'
        })
    </script>
</body>

优化valine.min.js加载

  1. 首先unpkg.com访问速度偏慢，直接将该文件copy到本地。

  2. 但这样运行后在国内仍无法访问，仔细看后台资源请求：

    红框里是影响国内用户访问速度的罪魁祸首，其他都是国内或CDN镜像的，不管他们。

    首先第二个红框是原js文件网址，说明原js中又重新引入了该网站另一个js文件。

    看看引用的是啥：

    在原js中找到它：

    将这个文件按第一步操作，copy到本地。

  3. 现在处理第一个红框，gravatar.loli.net其实是一个头像网站。全称Globally Recognized Avatar，翻译成中文为全球通用头像。

    就是提供评论者头像使用的，但服务器在国外，国内容易被墙。

    所以这里有两个办法：

    • 自己搭建一个镜像站。
    • 使用别人的镜像站。

    但好在Cravatar出现了，是中国的互联网头像服务站。100%兼容gravatar的同时还有以下优势：

    1. 更快的速度 – 服务完全架设并运行在中国大陆境内，提供毫秒级的响应速度
    2. 更高的稳定性 – 所有头像经人工审核确保不会出现违规内容，也就不会出现时不时访问不了的情况
    3. 更高的头像展现率 – 在自有头像库的基础上囊括了Gravatar和QQ的头像库，可谓集天下之大成

    所以直接将valine.min.js中使用的头像网站换成Cravatar即可。

后续

只是拿valine和gravatar做个例子，其他访问速度慢或被墙的问题也可以照这个思路解决。
