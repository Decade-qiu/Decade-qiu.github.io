Leetcode 1233. 删除子文件夹

字典树+排序+字符串

Leetcode

1233. 删除子文件夹 - 力扣（Leetcode）

你是一位系统管理员，手里有一份文件夹列表 folder，你的任务是要删除该列表中的所有 子文件夹，并以 任意顺序 返回剩下的文件夹。如果文件夹 folder[i] 位于另一个文件夹 folder[j] 下，那么 folder[i] 就是 folder[j] 的 子文件夹 。

文件夹的「路径」是由一个或多个按以下格式串联形成的字符串：'/' 后跟一个或者多个小写英文字母。

  • 例如，"/leetcode" 和 "/leetcode/problems" 都是有效的路径，而空字符串和 "/" 不是。

示例 1：

输入：folder = ["/a","/a/b","/c/d","/c/d/e","/c/f"]
输出：["/a","/c/d","/c/f"]
解释："/a/b" 是 "/a" 的子文件夹，而 "/c/d/e" 是 "/c/d" 的子文件夹。

示例 2：

输入：folder = ["/a","/a/b/c","/a/b/d"]
输出：["/a"]
解释：文件夹 "/a/b/c" 和 "/a/b/d" 都会被删除，因为它们都是 "/a" 的子文件夹。

示例 3：

输入: folder = ["/a/b/c","/a/b/ca","/a/b/d"]
输出: ["/a/b/c","/a/b/ca","/a/b/d"]

提示：

  • 1 <= folder.length <= 4 * 10^4
  • 2 <= folder[i].length <= 100
  • folder[i] 只包含小写字母和 '/'
  • folder[i] 总是以字符 '/' 起始
  • 每个文件夹名都是 唯一 的

思路

根据题意，容易判断文件结构是一个树形结构。具体目标就是找出这些文件夹的不同的最短前缀集合。

  1. 暴力思考。遍历每个文件，看有没有其他文件是它的前缀。如果没有就加入答案。

  2. 优化：越长的文件名称越不可能作为其他文件前缀，所以可以将文件夹长度从小到大排序。

    对于第 iii 个文件，只需查找前 i−1i-1i−1 个文件中有没有是它的前缀即可。

  3. 再次优化：由树形结构和前缀搜索可以想到字典树。当插入第 iii 个文件时，判断每一层是否已经是一个文件的结束层。

    如果是，说明前面插入的文件有它的前缀，就不再插入；反之，插入成功，加入答案。

代码

class Solution {
    T root;
    boolean add(String s){
        String[] ss = s.split("/");
        T cur = root;
        boolean ans = false;
        for (String i : ss){
            if (!cur.next.containsKey(i)){
                cur.next.put(i, new T());
                ans = true;
            }cur = cur.next.get(i);
            if (cur.end) return false;
        }cur.end = true;
        return true;
    }
    public List<String> removeSubfolders(String[] f) {
        root = new T();
        List<String> res = new ArrayList<>();
        Arrays.sort(f, (x,y)->x.length()-y.length());
        for (String s : f){
            if (add(s)) res.add(s);
        }
        return res;
    }
}
class T{
    boolean end = false;
    Map<String, T> next = new HashMap<>();
}

复杂度分析

  • 时间复杂度: O(nm)O(n m)O(nm) 。n,mn, mn,m分别是输入文件数组的长度和平均文件长度。
  • 空间复杂度： O(nm)O(nm)O(nm) 。
