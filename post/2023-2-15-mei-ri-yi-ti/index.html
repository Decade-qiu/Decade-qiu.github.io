Leetcode 1250. 检查「好数组」
CodeForces 1374 E1 选书

CodeForces

Problem - E1 - Codeforces

给你n本数，俩个人对这n本书有的喜欢，有的讨厌。现在问，选出任意数量的书，这些书至少有k个是第一个人喜欢的，至少有k个是第二个人喜欢的。现在问选出的书的阅读时间总和最少为多少(如果挑不出来，输出-1)?

input

8 4
7 1 1
2 1 1
4 0 1
8 1 1
1 0 1
1 1 1
1 0 1
3 0 0

output

18

思路

有四种书:
1.两个人都能拿。2.只有 A\mathrm{A}A 能拿。3.只有B能拿。4.都不能拿。

显然，选第四个是毫无意义的，只能从前三种选。

对于2，3 我们可以合起来考虑。因为要优先选第一个类型的，但如果第一个类型的阅读时间为100，同时有一个第二类型的为5，一个第三类型的为10。显然我们可以选择后两个来代替第一个类型，因为5+10 < 100。

所以，对2，3类型排序，然后每次从2，3中分别选最小的相加作为第一种类型，添加到第一种集合去。

最后对第一种排序，取前k个。如果不满k，输出-1.

代码

def solve():
    n, m = list(map(int, input().split()))
    a, b, c = [], [], []
    for i in range(n):
        t, x, y = list(map(int, input().split()))
        if x and y: c.append(t)
        elif x: a.append(t)
        elif y: b.append(t)
    a.sort()
    b.sort()
    for i in range(min(len(a), len(b))): c.append(a[i]+b[i])
    if len(c) < m:
        print(-1)
        return
    ans = 0
    c.sort()
    for i in range(m): ans += c[i]
    print(ans)

复杂度分析

  • 时间复杂度: O(nlog⁡n)O(n \log n)O(nlogn) 。
  • 空间复杂度： O(n)O(n)O(n) 。

1250. 检查「好数组」 - 力扣（Leetcode）

给你一个正整数数组 nums，你需要从中任选一些子集，然后将子集中每一个数乘以一个 任意整数，并求出他们的和。

假如该和结果为 1，那么原数组就是一个「好数组」，则返回 True；否则请返回 False。

示例 1：

输入：nums = [12,5,7,23]
输出：true
解释：挑选数字 5 和 7。
5*3 + 7*(-2) = 1

示例 2：

输入：nums = [29,6,10]
输出：true
解释：挑选数字 29, 6 和 10。
29*1 + 6*(-3) + 10*(-1) = 1

示例 3：

输入：nums = [3,6]
输出：false

提示：

  • 1 <= nums.length <= 10^5
  • 1 <= nums[i] <= 10^9

思路

从两个数入手，假设两个正整数 ppp 和 qqq 的最大公约数 为 ggg ，考虑任意整数 xxx 和 yyy ，存在 xp+yq=ykg+ztg=(xk+yt)gx p+y q=y k g+z t g=(x k+y t) gxp+yq=ykg+ztg=(xk+yt)g 。 又 (xk+yt)(x k+y t)(xk+yt) 也是整数，如果要找到某一对 (x,y)(x, y)(x,y) 使得 (xk+yt)g=1(x k+y t) g=1(xk+yt)g=1 成立， 由之前的分析可知唯一的可能性便是 g=1g=1g=1 ，也就是 ppp 和 qqq 的最大公约数必须是 1 。

代码

class Solution {
    public boolean isGoodArray(int[] nums) {
        int n = nums.length;
        int ans = nums[0];
        for (int i : nums) ans = gcd(ans, i);
        return ans==1;
    }
    int gcd(int x, int y){
        return x==0 ? y : gcd(y%x, x);
    }
}

复杂度分析

  • 时间复杂度: O(n+log⁡m)O(n+\log m)O(n+logm) ，其中 nnn 为数组 numsn u m snums 的长度， mmm 为数组 numsn u m snums 中的最大 数，其中求单次最大公约数的时间复杂度为 O(log⁡m)O(\log m)O(logm) ，由于在每次求两个数的最大公约 数时其中一个数保持单调不增，所以求总的公约数的时间复杂度为 O(log⁡m)O(\log m)O(logm) 。
  • 空间复杂度： O(1)O(1)O(1) 。仅使用常量空间。
