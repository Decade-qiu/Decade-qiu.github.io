1223. 掷骰子模拟 - 力扣（Leetcode）

有一个骰子模拟器会每次投掷的时候生成一个 1 到 6 的随机数。

不过我们在使用它时有个约束，就是使得投掷骰子时，连续 掷出数字 i 的次数不能超过 rollMax[i]（i 从 1 开始编号）。

现在，给你一个整数数组 rollMax 和一个整数 n，请你来计算掷 n 次骰子可得到的不同点数序列的数量。

假如两个序列中至少存在一个元素不同，就认为这两个序列是不同的。由于答案可能很大，所以请返回 模 10^9 + 7 之后的结果。

示例 1：

输入：n = 2, rollMax = [1,1,2,2,2,3]
输出：34
解释：我们掷 2 次骰子，如果没有约束的话，共有 6 * 6 = 36 种可能的组合。但是根据 rollMax 数组，数字 1 和 2 最多连续出现一次，所以不会出现序列 (1,1) 和 (2,2)。因此，最终答案是 36-2 = 34。

示例 2：

输入：n = 2, rollMax = [1,1,1,1,1,1]
输出：30

示例 3：

输入：n = 3, rollMax = [1,1,1,2,2,3]
输出：181

提示：

  • 1 <= n <= 5000
  • rollMax.length == 6
  • 1 <= rollMax[i] <= 15

思路

常规DP问题，状态转移方程比较容易看出来（🙌当然是不考虑状态表示优化的前提上）。

对于每次掷骰子，可以有0~5一共6个数字选择。考虑到有连续出现数字的限制，还需要记录每次掷骰子后，0~5连续出现的次数。

于是可以得到定义dp[i][j][k]:完成了i次掷骰子，第i次掷的是j，并且已经连续掷了k次j的合法序列数。

转移时，分两种情况讨论：

  • k = 1：说明第一次掷到 j ,加上前i-1次所有掷的不是 j 的合法方案即可。
  • k >= 2: 说明第i-1次必须掷的是 j ，加上 dp[i−1][j][k−1]dp[i-1][j][k-1]dp[i−1][j][k−1] 即可。

代码

class Solution {
    public int dieSimulator(int n, int[] rollMax) {
        int M = (int)(1e9+7), ans = 0;
        int[][][] dp = new int[n+1][6][16];
        for (int i = 0;i < 6;i++) dp[1][i][1] = 1;
        for (int i = 2;i <= n;i++){
            for (int j = 0;j < 6;j++){
                for (int t = 0;t < 6;t++){
                    for (int k = 1;k <= rollMax[t];k++){
                        if (t != j){
                            dp[i][j][1] = (dp[i][j][1]+dp[i-1][t][k])%M;
                        }else if (k > 1){
                            dp[i][j][k] = (dp[i][j][k]+dp[i-1][t][k-1])%M;
                        }
                    }
                }
            }
        }
        for (int i = 0;i < 6;i++){
            for (int j = 1;j <= rollMax[i];j++){
                ans = (ans + dp[n][i][j])%M;
            }
        }
        return ans;
    }
}

复杂度分析

  • 时间复杂度：O(nm2k)O(n m^2 k)O(nm2k)。其中 nnn 是掷骰子的次数，mmm 是随机数的种类数，kkk 是 rollMax 数组中的最大值。
  • 空间复杂度： O(nmk)O(n m k)O(nmk) 。
